1. 异常概述与异常体系结构

  • 异常：

在系统运行过程中会遇到一些问题，这类问题在开发代码中难以避免，例如：客户输入数据的格式，读取文件是否存在等。注意：在程序执行中发生，开发过程中的语法错误和逻辑错误不是异常。

  • 异常处理：使得程序可以处理非预期的情形，并继续正常的运行。

  • 产生异常的原因：
    1. Java内部错误产生异常。例如：Java虚拟机异常、资源耗尽
    2. 编程错误。例如：空指针访问、数组越界
    3. throw语句手动生成异常。一般用来告知该方法的调用者一些必要信息

  • 异常体系结构：

    为了能够及时有效地处理程序中的运行错误，Java 专门引入了异常类。在 Java 中所有异常类型都是内置类 java.lang.Throwable 类的子类。

    • 两个异常子类：Error、Exception
      • Error异常类：重大灾难！一般不编写针对性的代码进行处理。
      • Exception异常类：我们这章节讨论的重点。又分为Unchecked Exception（运行时/非受检/不检查异常）和Checked Exception（非运行时/受检/检查异常）
    • Unchecked Exception 与 Checked Exception
      • Unchecked Exception（运行时/非受检/不检查异常）：都是Runtime异常，可以不做处理。这些异常一般由程序逻辑错误引起，程序应该从逻辑角度尽可能避免这类异常的发生。
      • Checked Exception（非运行时/受检/检查异常）：编译器要求必须处理的异常，一般是程序在运行时由于外界因素造成的一般性异常

  • 对于错误的解决方法：

    错误都是要处理的。上面说Unchecked Exception可以不做处理，是指程序猿可以不编写代码对它进行处理。

    1. 遇到错误就终止程序的运行。
    2. 程序猿编写程序时，就考虑到错误的检测、错误消息的提示、以及错误的处理。

3. 异常处理机制一：try-catch-finally

01.异常与异常处理

  • 异常是一种对象（这点的理解非常重要），表示阻止正常进行程序执行的错误。在Java中，运行时错误会作为异常抛出。如果异常没有被处理，程序会非正常终止。

  • 异常处理的原理：==异常是从方法抛出的，方法的调用者可以捕获以及处理该异常。==

      public class QuotientWithMethod { 
  public static int quotient(int number1, int number2) {
    if (number2 == 0) {
      System.out.println("Divisor cannot be zero");
      System.exit(1);		
    }
    
    return number1 / number2;
  }
      
  public static void main(String[] args) {
    Scanner input = new Scanner(System.in);
        
    // Prompt the user to enter two integers
    System.out.print("Enter two integers: ");
    int number1 = input.nextInt();
    int number2 = input.nextInt();
        
    int result = quotient(number1, number2);
    System.out.println(number1 + " / " + number2 + " is " 
      + result);
  }
}
      • 分析：
        • exit(1)表示异常终止、exit(0)表示正常终止，exit终止整个程序。
        • 分析：如果number2为0，程序会在第10行终止。
        • 则显然是个问题，‼️==不应该让方法来终止程序 —— 一个由调用者决定是否终止程序。==

    • 方法如何通知它的调用者一个异常产生了呢？ —— Java可以让一个方法抛出一个异常，该异常可以被调用者捕获和处理。
