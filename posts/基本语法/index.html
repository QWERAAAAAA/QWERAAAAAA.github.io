1. Java语言概述

  • Java是一种面向Internet的编程语言。Java applet可以在Web浏览器中运行。applet内嵌在HTML代码中。
  • Java VS C++ ：
    • Java舍弃了C中容易引起错误的指针（以引用取代）、运算符重载、多重继承（以借口取代）
    • ==增加了垃圾回收器功能。==（仍会出现内存泄漏和内存溢出的问题）
  • Java是体系结构中立的。Java程序（后缀为java）可以在Java平台被编译为体系结构中立的字节码格式（后缀为class），然后可以在实现这个Java的平台的任何系统中运行。
  • Java语言是解释型的。

01 Java虚拟机JVM

“Write once，Run Anywhere” —— 跨平台性 —— 由Java虚拟机（JVM）实现。

02 Java语言环境

JDK、JRE、JVM三者关系。

  • JDK = JRE + 开发工具集（例如Javac编译工具等）
  • JRE = JVM + Java SE标准类库

分析：

  • 每个Java程序都是一套类的定义集合。Java程序中的全部内容都必须放在类中。（强制面向对象）
  • 方法包含在类中，每个可执行的Java程序必须有一个main方法，main方法是程序开始执行的入口。

  1. 将Java代码编写到扩展名为.java 的文件中。

  2. 通过javac命令对该Java文件进行编译。

    ​ javac HelloWorld.java

  3. 通过java命令对生成的class文件进行运行。

​ java HelloWorld

3. 标识符

凡是可以自己起名字的地方都叫做标识符。

规范：

  • 包名：字母全部小写。xxxyyyzzz
  • 类名、接口名：所有首字母大写。XxxYyyZzz
  • 变量名、方法名：多单词时，第一个首字母大写，之后的首字母大写。xxxYyyZzz
  • 常量名：所有字母大写，多单词用下划线连接。XXX_YYY_ZZZ

变量是程序中最基本的存储单元。包括变量类型、变量名和存储的值。

变量用于表示在程序中可能被改变的值。

03 基本数据类型变量的转换

自动类型转换：容量小的类型自动转换为容量大的数据类型。

注意：

  1. boolean类型不能与其它数据类型运算
  2. 当把任何基本数据类型的值和字符串(String)进行连接运算时(+)，==基本数据类型的值将自动转化为字符串(String)类型。==

强制类型转换：通常是将容量大的数据类型转换为容量小的数据类型

使用时要加上强制转换符：( )，但可能造成精度降低或溢出,格外要注意。

  • 通常，字符串不能直接转换为基本类型，但通过基本类型对应的包装类则可以实现把字符串转换成基本类型。
  • 如： String a = “43”; int i = Integer.parseInt(a);

⚠️ 下面的代码是正确的

1
2
3

int sum = 0;
sum += 4.5;
// 等价于sum = (int)(sum + 4.5)， sum的结果为4

5. 常量

常量有三种类型：静态常量、成员常量和局部常量。

定义常量—— final关键字

public class HelloWorld {
    // 静态常量PI
    public static final double PI = 3.14;
    
  	// 声明成员常量y
    final int y = 10;

    public static void main(String[] args) {
        
      	// 声明局部常量x
        final double x = 3.3;
    }
}
  • 静态常量PI：使用在 final 之前 public static 修饰。public static 修饰的常量作用域是全局的，不需要创建对象就可以访问它，在类外部访问形式为 HelloWorld.PI
  • 成员常量y：作用域类似于成员变量，但不能修改。（注意：与静态成员static int y = 10;不一样。
  • 局部常量x：作用域类似于局部变量，但不能修改。

在定义常量时，需要注意如下内容：

  • 在定义常量时就需要对该常量进行初始化。
  • final 关键字不仅可以用来修饰基本数据类型的常量，还可以用来修饰对象的引用或者方法。
  • 为了与变量区别，常量取名一般都用大写字符。

当常量被设定后，一般情况下不允许再进行更改，如果更改其值将提示错误。

  • 例如：+=、-=、*=、/=、%=

    等价于：

    1

    x = x / (4 + 5.5 * 1.5);

8. 程序流程控制

流程控制语句是用来控制程序中各语句执行顺序的语句，可以把语句组合成能完成一定功能的小逻辑模块。

01 顺序结构

程序从上到下逐行地执行，中间没有任何判断和跳转。

程序是从上到下逐行执行的，所有语句的先后次序，有时对程序的执行也起到了很大的影响。尤其是在循环体和选择结构中！

02 分支结构

根据条件，选择性地执行某段代码。 有if…else和switch-case两种分支语句。（这个玩法就多样了！）

if结构：三种玩法供你选择！

✍🏼if-else if-else的技巧：

  • 当多个条件是“互斥”关系时，条件判断语句及执行语句间顺序无所谓

  • 当多个条件是“包含”关系时，“小上大下 / 子上父下”。（这个if中的条件不满足才会往下一个去）
