## 1.继承的目的

继承提供了在已有类的基础上开发出新类的机制，可以**节省重复代码的编写**工作，是软件重用的基础。





## 2.继承的概念

1⃣️ 已知一个类A，对A类加以扩展，即**增加一些属性和行为**，构成一个新类B，此时**B类将A类已有的属性和行为继承下来**。称类 B 继承了 A 或称类 A 派生了 B 。

​	A ：父类，基类

​	B ：子类，派生类

### 📝 单一继承

定义派生类的一般格式：

```
class <派生类名>:[<继承方式>]<基类名>{

	// 定义派生类新成员
	
	private:		// 私有成员
	public:			// 公有成员
	protected:	// 保护成员
	
};
```



### 📝 继承方式

1. 继承方式有三种：

​		public —— 公有继承

​		private —— 私有继承

​		protected —— 保护继承



2. 不同继承方式下，基类成员的访问权限在派生类中的变化：

<img src="https://tva1.sinaimg.cn/large/008vxvgGgy1h7m3a1h4aej30ni08ygmd.jpg" alt="截屏2022-10-29 13.22.45" style="zoom:67%;" />

​	分析可知：

- **基类成员在派生类中的访问权限不得高于继承方式中指定的权限。**

- 不管继承方式如何，**基类中的 private 成员在派生类中始终不能==直接==使用（不能在派生类的成员函数中访问或调用）**。



#### 🔎 何为不可见

> 我们这里说的是基类的 private 成员不能在派生类中直接使用，并没有说基类的 private 成员不能被继承。

​	实际上，**基类的 private 成员是能够被继承的，并且（成员变量）会占用派生类对象的内存**，它只是在派生类中不可见，导致无法使用罢了。（可以使用基类的公有函数对其进行访问）

​	private 成员的这种特性，能够很好的对派生类隐藏基类的实现，以体现面向对象的封装性。







## 3.公有继承

> 公有继承是真正意义上的继承。

```
class A{
	private: int ax;
	protected: int ay;
	public: 
		int az;
		void f1(){...};
		int Getax(){return ax;}
		void Setax(int x){ax=x};
};

class B: public A{
	private: int bx;
	protected: int by;
	public:
		int bz;
		void f2(){...};
		float f3(){...};
};
```

## 4.protected成员在继承上的优势



对于一个单独的类来说，private成员和protected成员都是在类内可以直接访问的，在类外不可以直接访问，在这方面没有什么区别。

在继承的情况下，private成员都无法在派生类中直接访问。而**protected成员可以在派生类中直接访问**。——优势



**protected成员的优点：**

- 既可以在本类中实现数据的隐藏（类内可直接访问、类外不可被直接访问）
- 又可以将其类内直接访问特性传递到派生类中（在派生类内可直接访问、类外不可被直接访问）





## 5.多重继承

> 一个派生类有多个基类。

定义的一般格式：

```
class<派生类名>: <继承方式1><基类名1>,<继承方式2><基类名2>{
	private:
	protected:
	public:
};
```
