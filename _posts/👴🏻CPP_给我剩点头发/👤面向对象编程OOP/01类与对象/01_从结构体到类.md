# 01 从结构体到类

任何事物都可以称为对象，每一个对象都具有静态属性和动态属性。

一般将对象的数值属性称为静态属性，将对象可进行的动作称为动态属性。

▪️结构体：**描述对象的静态属性**抽象为结构体类型，是一种导出数据类型。

```C++
struct SPerson{
	char name[20];
	char sex;
	int age;
	// 三个数据成员
}
```

▪️类：把对象可进行的操作也加入到对象的描述中，是对具有相同属性和行为的一个或多个对象的描述。

🔹类和对象定义格式：

```C++
class <类名>{
	private:		// 可省略
		<数据成员及成员函数>
	public:
		<数据成员及成员函数>
};
```

🔹一个简单的类的定义：

```C++
// 建立头文件person.h
class Person{

	// 成员变量,默认为私有
	char Name[20];
	char Sex;
	int Age;
	
	// 成员函数,在类内声明
	public:
		void SetData(char na[], char s, int a);
		void DetName(char *na);
		char GetSex();
		int GetAge();
};

// 在类外定义成员函数
void Person::SetData(char na[], char s, int a){
  strcpy(Name, na);
  Sex = s;
  Age = a;
}
void Person::GetName(char *na){
  strcpy(na, Name);
}
void Person::GetSet(){
  return Sex;
}
void Person::GetAge(){
  return Age;			// 注意return不是cout，不会输出，但会返回给调用者，之后的操作由调用者决定。
}
```

🔺类只是一个模板（Template），编译后不占用内存空间，所以在定义类时**不能对成员变量进行初始化**，因为没有地方存储数据。只有在创建对象以后才会给成员变量分配内存，这个时候就可以赋值了。

🔺类内定义（内联函数）一般不是我们所期望的，它会将函数调用处用函数体替代，所以我建议在类体内部对成员函数作声明，而在类体外部进行定义，这是一种良好的编程习惯，实际开发中大家也是这样做的。

🔺在类体中直接定义函数时，不需要在函数名前面加上类名，因为函数属于哪一个类是不言而喻的。但当成员函数定义在类外时，就必须在函数名前面加上类名予以限定。`::`被称为**域解析符（也称作用域运算符或作用域限定符）**，用来连接类名和函数名，指明当前函数属于哪个类。

🔺类的声明和成员函数的定义都是类定义的一部分，在实际开发中，我们通常**将类的声明放在头文件中，而将成员函数的定义放在源文件中**。

🔺成员变量大都以`m_`开头，这是约定成俗的写法，不是语法规定的内容。**以`m_`开头既可以一眼看出这是成员变量，又可以和成员函数中的形参名字区分开**。



▪️创建对象：

格式：`<类名> <对象列表>;`

```C++
Person a, b;		// 定义了a、b两个对象
类似 int a;			// 定义整型变量
```

我们可以把Person看做一种新的数据类型。

也可以定义对象数组

```C++
Person allStud[100];
```

创建了一个allStud数组，它有100个元素，每个元素都是Person类型的对象。



▪️对象成员的访问：

Person类的三个成员对象都是私有的，在类内可直接访问它们。

“类内”指在类的成员函数中，“类外”指在类的成员函数之外（通过对象名、对象指针不可访问对象成员）

优点：类中的数据成员通常私有，在类外不可直接访问，达到“保护”/信息隐蔽作用。

```C++
a.Name、a.Sex、pa->Name是非法的
a.GetName()、a.GetSex()、pa->SetData()合法
```

可以把类看成“黑箱子”，私有成员被“隐藏”在黑箱子中，要想存取私有成员，必须通过黑箱子上的专用窗口（即公有成员函数接口）来进行。



▪️使用对象指针

> 直接定义对象是在栈内存上分配内存，new出来的对象是在堆上分配内存

```C++
Person stu;
Person *p = &stu;			// 创建的对象stu在栈上分配内存，需要使用&获取它的地址
```

p是一个指针，指向Person类型的数据，也是通过Person创建出来的对象。

可以在堆上创建对象，这时候需要用到new关键字

```c++
Person *p = new Person;
```

🔺通过new创建出来的对象，在堆上分配内存，没有名字，只能得到一个指向它的指针，所以必须使用一个指针变量来接收这个指针，否则以后再也无法找到这个对象。

🔺栈内存是程序自动管理的，不能使用delete删掉在栈上创建的对象；堆内存由程序员管理，对象使用完毕后可以通过 delete 删除。在实际开发中，new 和 delete 往往成对出现，以保证及时删除不再使用的对象，防止无用内存堆积。

有了对象指针之后，可以通过箭头->来访问对象的成员变量和成员函数

```c++
p->SetData("Zhang", 'M', 18);
```



▪️测试Person类：

```c++
#include<iostream>
#include<cstring>
#include"Person.h"
using namespace std;
int main(){
	Person a, *pa;
	char name[20];
	
	// 通过对象访问成员
	a.SetData("Cheng", 'F', 20);
	a.GetName(name);	
	cout << "Name:" << name << '\t';
	cout << "Sex:" << a.GetSex() << '\t';		// cout把return的返回值输出出来。
	
	// 通过指针访问成员
	pa = &a;
	pa->SetData("Zhang", 'M', 18);
	pa->GetName(name);
	cout << "Name:" << name << '\t';
	cout << "Sex:" << pa->GetSex() << '\t';
	return 0;
}
```



▪️对象的存储空间

> 用类定义的对象所占的存储空间

​	系统将成员函数的存储空间处理成该类的所有对象共享同一代码空间。因此一个对象的存储空间是对象自身数据成员的存储空间。（不对成员函数单独分配空间）

​	对象存储空间的分配和撤销是系统根据对象的作用域自动完成的；当使用new和delete运算符动态建立和撤销对象时，对象存储空间的分配和撤销是在new和delete执行时刻完成的。



▪️定义类和对象的有关说明

-  类的数据成员的类型可以是：基本数据类型、构造数据类型、已经定义的类、自身类的指针或引用（不是一个实际对象）。

- 定义类的对象有三种方法：

  1. 定义完类后，定义对象

     `class Date{...};	Date d1; `

  2. 在定义类的同时定义对象

     `class Date{...} d1,d2,d3;`

  3. 定义无类名的对象

     `class {...} d1,d2,d3;`
