## 1. 异常概述与异常体系结构

- **异常**：

在系统运行过程中会遇到一些问题，这类问题在开发代码中难以避免，例如：客户输入数据的格式，读取文件是否存在等。**注意**：在程序执行中发生，开发过程中的语法错误和逻辑错误不是异常。

- **异常处理：**使得程序可以处理非预期的情形，并继续正常的运行。



- **产生异常的原因：**
  1. Java内部错误产生异常。例如：Java虚拟机异常、资源耗尽
  2. 编程错误。例如：空指针访问、数组越界
  3. throw语句手动生成异常。一般用来告知该方法的调用者一些必要信息



- **异常体系结构：**

  > 为了能够及时有效地处理程序中的运行错误，**Java 专门引入了异常类**。在 Java 中所有异常类型都是内置类 `java.lang.Throwable 类`的子类。

  <img src="https://tva1.sinaimg.cn/large/008vxvgGgy1h803yhlue5j30c506zdfx.jpg" alt="img" style="zoom:67%;" /><img src="https://tva1.sinaimg.cn/large/008vxvgGgy1h8043k8e0nj30s10rswhr.jpg" alt="IMG_8DE756ADF787-1" style="zoom: 33%;" />

  - **两个异常子类：Error、Exception**
    - Error异常类：重大灾难！一般不编写针对性的代码进行处理。
    - Exception异常类：我们这章节讨论的重点。又分为**Unchecked Exception**（运行时/非受检/不检查异常）和**Checked Exception**（非运行时/受检/检查异常）
  - **Unchecked Exception 与 Checked Exception**
    - **Unchecked Exception**（运行时/非受检/不检查异常）：都是Runtime异常，**可以不做处理**。这些异常一般由**程序逻辑错误引起**，程序应该从逻辑角度尽可能避免这类异常的发生。
    - **Checked Exception**（非运行时/受检/检查异常）：**编译器要求必须处理的异常**，一般是程序在运行时由于外界因素造成的一般性异常



- **对于错误的解决方法：**

  **错误都是要处理的。**上面说Unchecked Exception可以不做处理，是指程序猿可以不编写代码对它进行处理。

  1. 遇到错误就终止程序的运行。
  2. 程序猿编写程序时，就考虑到错误的检测、错误消息的提示、以及错误的处理。



## 2. 常见异常

- ⬇️ 常见Unchecked异常（Runtime异常）

| 异常类型                          | 说明                                                  |
| --------------------------------- | ----------------------------------------------------- |
| **ArithmeticException**           | 算术错误异常，如以零做除数                            |
| **ArraylndexOutOfBoundException** | 数组索引越界                                          |
| ArrayStoreException               | 向类型不兼容的数组元素赋值                            |
| ClassCastException                | 类型转换异常                                          |
| IllegalArgumentException          | 使用非法实参调用方法                                  |
| lIIegalStateException             | 环境或应用程序处于不正确的状态                        |
| lIIegalThreadStateException       | 被请求的操作与当前线程状态不兼容                      |
| IndexOutOfBoundsException         | 某种类型的索引越界                                    |
| **NullPointerException**          | 尝试访问 null 对象成员，空指针异常                    |
| NegativeArraySizeException        | 再负数范围内创建的数组                                |
| NumberFormatException             | 数字转化格式异常，比如字符串到 float 型数字的转换无效 |
| TypeNotPresentException           | 类型未找到                                            |

## 3. 异常处理机制一：try-catch-finally



### 01.异常与异常处理

**异常是一种对象**（这点的理解非常重要），表示阻止正常进行程序执行的错误。在Java中，运行时错误会作为异常抛出。如果异常没有被处理，程序会非正常终止。

**异常处理的原理：**异常是从方法抛出的，方法的调用者可以捕获以及处理该异常。
