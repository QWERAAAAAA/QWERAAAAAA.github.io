🤔贪心算法和动态规划有许多相似之处，写题时如何选择？来看两个经典的组合优化问题。



## 0-1背包问题 & 背包问题

👇🏻还是简单的描述一下问题：

### 0-1背包问题

给定n种物品和一个书包，物品i的重量为wi，价值为vi，背包容量为c。

在选择装入背包的物品时，只有两种选择：装入或不装入。

问：应如何选择装入背包中的物品，使得装入背包中物品的总价值最大？



![IMG_76997083C464-1](https://tva1.sinaimg.cn/large/008vxvgGgy1h7wm5hmgcvj33340c3tc9.jpg)



### 背包问题

和0-1背包问题类似，区别在：选择物品i（1 <= i <= n）装入背包时，可以选择物品i的一部分，而不一定要全部转入背包。



![IMG_D39EC19770FA-1](https://tva1.sinaimg.cn/large/008vxvgGgy1h7wma4uh2gj33340c2tc8.jpg)



`✍🏼背包问题可以用贪心算法求解，而0-1背包问题不能用贪心算法求解。`



### 贪心算法求解背包问题

求解的基本步骤：

1. 计算每种物品**单位重量的价值**vi/wi
2. **设计贪心选择策略**：先装单位价值**最高**的，再装单位价值**次高**的，以此推进下去，直到背包装满为止。

```C++
void Knapsack(int n, float M, float v[], float w[], float x[]){
	Sort(n,v,w);										// 将单位价值进行排序
  int i;
  for(i = 1; i <= n; i++)
    	x[i] = 0;										// x[]是一个n元向量，0表示未装入，1表示装入
  float c = M;										// c为背包容量
  for(i = 1; i <= n; i++){
    if(w[i] > c)
      break;
    x[i] = i;
    c -= w[i];
  }
  if(i <= n)
    x[i] = c/w[i];		
}
```

`n为物品品种数， x[i]表示背包中某种物品的质量，所以x[n+1]表示最终的最优解。`



### 贪心算法求解0-1背包问题

**当然是不行啦！**

🌰举个栗子：

![IMG_78B0193B20FD-1](https://tva1.sinaimg.cn/large/008vxvgGgy1h7wn837gooj329q0rsdkv.jpg)

三个物品，背包容量为50kg，物品1重10kg，价值60RMB，物品2重20kg，价值100RMB，物品3重30kg，价值120RMB。

单位价值：物品1 > 物品2 > 物品3

按照贪心选择策略，首选物品1装入背包，但是由(b)图可知，最优的情况是选择物品2和物品3装入。

而对于(c)图，在背包算法的情况下，贪心选择可以得到最优解。



**原因：**

- 0-1背包问题，**贪心选择无法保证最终能将背包装满**，部分闲置的背包空间**拉低了每千克背包空间的价值**。

- 在考虑0-1背包问题时，应比较选择该物品和不选择该物品所导致的最终方案，再作出选择。而我们说过，“**选择的贪心策略必须具备`无后效性`，即某个状态以前的过程不会影响以后的状态，只与当前状态有关。**”所以不适合用贪心算法，DP算法就可以有效的解决0-1背包问题。





## 最优装载问题

👇🏻简单的描述一下问题：

一批集装箱要装上一艘载重为c的轮船。其中集装箱i的重量为wi。问：如何使得尽可能多的集装箱上船。

![IMG_8E0AE781DC7E-1](https://tva1.sinaimg.cn/large/008vxvgGgy1h7wnwflphjj32f40rs0x4.jpg)



### 贪心算法解决最优装载问题

这个问题是不是和0-1背包问题很像，但是**为什么它可以用贪心算法**呢？

它**和0-1背包问题的差别**在于：我不考虑物品装包后的价值如何，只考虑能否把它尽可能多的都装包，当然，先装最轻的就可以装最多啦～

所以，采用**重量最轻者先装**的贪心选择策略。

```C++
template<class Type>
void Loading(int x[], Type c, int n){
	int *t = new int [n+1];
  Sort(w, t, n);
  for(int i = 1; i <= n; i++)
    x[i] = 0;
  for(int i = 1; i <= n && w[t[i]] <= c; i++){
    x[t[i]] = i;
    c -= w[t[i]];
  }
}
```
